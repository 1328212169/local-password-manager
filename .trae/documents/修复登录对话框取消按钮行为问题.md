## 问题分析

当软件处于「未设置主密码」初始状态时，用户点击登录页的「取消登录」按钮，然后在退出确认窗口中点击「取消」按钮，软件会直接退出，而不是跳转到「设置主密码」页面。

### 根因

1. 当用户在登录对话框中点击「取消登录」按钮时，会调用 `handle_login_cancel(login_dialog)` 方法
2. 在该方法中，当用户在退出确认窗口中点击「取消」按钮时，会检查 `self.master_password` 是否为空
3. 如果 `self.master_password` 为空，会显示设置主密码对话框
4. 当用户设置了主密码后，代码会调用 `login_dialog.accept()`，但是不会显示主窗口
5. 当 `show_login_dialog()` 方法结束后，会回到 `__init__` 方法的调用处，但是主窗口不会显示，因为 `handle_login_cancel()` 方法中没有显示主窗口的代码

## 解决方案

修改 `handle_login_cancel()` 方法，在用户设置了主密码后，不仅要调用 `login_dialog.accept()`，还要显示主窗口，就像 `login()` 方法一样。

## 具体修改

1. 在 `handle_login_cancel()` 方法中，当用户设置了主密码后，添加显示主窗口的代码：
   ```python
   if result == QDialog.DialogCode.Accepted:
       login_dialog.accept()
       # 强制显示主窗口并激活到前台
       self.showNormal()
       self.activateWindow()
       self.raise_()
       # 确保窗口状态正确
       self.setWindowState(Qt.WindowState.WindowActive)
       self.setFocus()
   ```

2. 保持其他逻辑不变，确保「确定」按钮的逻辑仍然是彻底退出软件

## 预期效果

现在，当软件处于「未设置主密码」初始状态时：
1. 打开软件 → 登录页点击「取消登录」→ 退出确认框点击「取消」
   - 软件不会退出，进程不会终止
   - 关闭退出确认框
   - 立即显示「设置主密码」页面
   - 设密码页面可正常输入密码、点击保存，无功能异常
   - 设置密码成功后会显示主窗口

2. 打开软件 → 登录页点击「取消登录」→ 退出确认框点击「确定」
   - 软件仍然会彻底退出（保持原有逻辑不变）
